# CLAUDE.md

## General principles
- This is scientific coding so it should fail fast with clear error messages if anything unexpected happens, do not silently pass over errors. For instance, when getting from dictionaries don't use `.get` with defaults unless that is specifically desired; instead raise an error if key does not exist.

- All input data go in `./data` and all results generated by the pipeline are placed in `./results`, except for the final generated trees that go in `./auspice`

- This is a Snakemake pipeline with the configuration in `config.yaml`, the conda environment in `environment.yaml`, the pipeline itself in `Snakefile`, and scripts in `./scripts`. The `config.yaml` uses [YTE](https://yte-template-engine.github.io/) templating (enabled by `__use_yte__: true`) with a `?for` loop to generate per-subtype tree configs from a single template.

- Keep the code as concise as possible

- Do not duplicate code in scripts; define things (eg, lists of expected columns) in one place.

- Keep this `CLAUDE.md` file updated after each change; also keep `README.md` updated with a more high level description.

## Input data organization
All input data is in `./data/`, documented in detail in [data/README.md](data/README.md). Key subdirectories:
- `data/auspice_config/`: auspice configuration files (auspice config JSON and description markdown) shared across trees
- `data/mutation_effect_data/`: raw experimental CSV files with mutation effect measurements (one per experiment)
- `data/trees/{tree}/`: per-tree input files (`reference_sequence.fa`, `annotation.gff`, `accessions_to_exclude.txt`, `accessions_to_include.txt`, `protein_sites.tsv`, `mutation_effects.tsv`)
- `data/strain_token_host_map.tsv`: shared across trees; maps lowercased 2nd `/`-delimited strain tokens to NCBI host tax IDs (used by `infer_missing_host` to fill host info when NCBI metadata is blank)

The `protein_sites.tsv` and `mutation_effects.tsv` files are **not manually created**; they are generated by `data/build_protein_sites_and_mutation_effects.py` (run manually, not part of the Snakemake pipeline) from the raw experimental CSVs, reference sequences, and the config in `data/build_protein_sites_and_mutation_effects_config.yaml`. This script aligns all reference proteins and experiment proteins, computes H3 numbering, and maps mutation effects across subtypes via the alignment. See `data/README.md` for full details.

## Code linting and formatting
After every change, run `snakefmt`, `snakemake --lint`, `black`, and `ruff` for formatting and linting.

## Pipeline rules

### `get_ncbi_dataset_zip`
Downloads all Influenza A virus sequences from NCBI using `datasets download virus genome taxon` (taxon ID in `config.yaml`). Includes genome, CDS, annotation, and biosample data. Outputs a zip file and a datestamp recording the download date.

The zip contains:
- `data_report.jsonl`: per-record metadata (accession, virus name with subtype, segment number, collection date, location, host, completeness)
- `annotation_report.jsonl`: per-gene annotation with gene name (e.g., "HA"), CDS name, and CDS FASTA sequence IDs
- `cds.fna`: nucleotide CDS FASTA with both full CDSs (e.g., "hemagglutinin") and mature peptide fragments (HA1, HA2, sig_peptide). Headers have format `>accession:start-end name [polyprotein=...] [organism=...]`
- `genomic.fna`: full segment nucleotide sequences
- `biosample_report.jsonl`: BioSample metadata

To parse full-length HA CDSs by subtype downstream:
- Filter `cds.fna` for full CDS entries (name "hemagglutinin", not HA1/HA2/sig_peptide fragments)
- Use `data_report.jsonl` field `segment` ("4" = HA) and `virus.organismName` or `virus.lineage` for subtype
- Use `annotation_report.jsonl` field `gene-name` ("HA") and `gene-cds-nuc-fasta-seq-id` to match CDS FASTA entries
- `dataformat tsv virus-genome --package <zip>` and `dataformat tsv virus-annotation --package <zip>` convert reports to TSV

### `format_ncbi_dataset`
Processes the NCBI zip once into TSV intermediates. Uses `dataformat tsv virus-genome` to extract genome metadata to `results/ncbi_dataset/genome_metadata.tsv` (includes `host-name` as NCBI Taxonomy scientific name and `host-tax-id` as the corresponding NCBI Taxonomy ID), extracts FASTA header lines from `genomic.fna` to `results/ncbi_dataset/genomic_headers.txt` (used to parse subtypes, which are only in the FASTA headers), and uses `dataformat tsv virus-annotation` to extract per-gene annotation (accession and gene name) to `results/ncbi_dataset/annotation_genes.tsv` (used to identify HA accessions that lack a segment annotation).

### `extract_ha_cds`
Per-subtype rule (wildcard `{tree}`) that extracts full-length HA CDS sequences and metadata. Uses `scripts/extract_ha_cds.py`. Identifies HA accessions as those with segment 4 in the genome metadata OR gene-name in {"HA", "ha", "HA1"} in the annotation genes TSV (some older NCBI records lack the segment field but have the gene annotation). Joins with subtypes parsed from genomic FASTA headers, filters by the `subtype` regex in `config["trees"][tree]`, then extracts matching full-length hemagglutinin CDS entries (not HA1/HA2/sig_peptide fragments) from `cds.fna` in the zip. For accessions with missing collection dates, the script attempts to parse a four-digit year from the strain name (e.g., "1930" from "A/swine/Iowa/15/1930") as a fallback. Sequences with ambiguous nucleotides (not ACTG) are dropped, sequences that fail Biopython `translate(cds=True)` are dropped, and sequences outside the CDS length range are dropped. The length range is read from a YAML file specified by `config["trees"][tree]["cds_length_range"]`, which has a `cds_length_range` key with a two-element list `[min, max]`; either bound can be `null` to impose no limit on that side. Outputs are sorted by date then strain. Output files are suffixed `_all` to indicate they are before any subsampling.

Outputs per tree:
- `results/trees/{tree}/metadata_all_pre_host.tsv`: columns are `accession`, `strain`, `subtype`, `date`, `host`, `host_tax_id`, `location`, `region`, `passage_history`, `length`
- `results/trees/{tree}/cds_all.fasta.gz`: gzipped full-length HA CDS nucleotide FASTA keyed by accession
- `results/trees/{tree}/extract_cds_stats.txt`: extraction statistics (subtype counts, filtering counts, CDS length distribution)

### `download_taxonomy`
Downloads NCBI taxonomy dump files (`taxdump.tar.gz`) from NCBI FTP for use by `taxonkit`. Extracts `names.dmp`, `nodes.dmp`, `delnodes.dmp`, and `merged.dmp`. Outputs to `results/ncbi_taxonomy/` directory. This is a shared resource used by all per-tree `annotate_host_taxonomy` rules.

Outputs:
- `results/ncbi_taxonomy/`: directory containing NCBI taxonomy dump files

### `merge_manual_add`
Per-subtype rule that merges manually-added CDS sequences and metadata (from `config["trees"][tree]["manual_adds"]["sequences"]` and `config["trees"][tree]["manual_adds"]["metadata"]`) into the Genbank-extracted outputs from `extract_ha_cds`. Uses `scripts/merge_manual_add.py`. Raises an informative error if: the manual-add metadata columns differ from the Genbank pre-host metadata columns; the sets of accessions in the manual-add FASTA and metadata disagree; any manual-add accession is duplicated within manual-add; or any manual-add accession contains whitespace. Accessions in manual-add that also appear in the Genbank data are dropped from the Genbank data (logged). Manual-add rows are appended after the Genbank rows. Empty manual-add inputs (header-only TSV and empty FASTA) pass through cleanly.

Outputs per tree:
- `results/trees/{tree}/metadata_all_pre_host_merged.tsv`: merged pre-host metadata (same columns as `metadata_all_pre_host.tsv`)
- `results/trees/{tree}/cds_all_merged.fasta.gz`: merged gzipped HA CDS FASTA

### `build_include_file`
Per-subtype rule that writes `results/trees/{tree}/accessions_to_include_combined.txt` as the sorted union of the user-maintained `config["trees"][tree]["accessions_to_include"]` (i.e. `data/trees/{tree}/accessions_to_include.txt`) and every accession in `config["trees"][tree]["manual_adds"]["metadata"]`. Uses `scripts/build_include_file.py`. The combined file is injected into `augur subsample --include` and `augur refine --keep-ids` by the Snakefile (the config itself does not reference any `results/` path).

### `infer_missing_host`
Per-subtype rule that fills `host` and `host_tax_id` for rows where both are empty by looking up the 2nd `/`-delimited token of the strain name (lowercased, whitespace-stripped) in `config["strain_token_host_map"]` (default `data/strain_token_host_map.tsv`). Uses `scripts/infer_missing_host.py`. The map has columns `token`, `host_tax_id`, `host_scientific_name`, `notes`; rows with blank `host_tax_id` are explicit "leave-empty" markers (locations, environmental samples, processed products, ambiguous terms like "turkey"). Validates the map: columns must match, tokens must be unique and already normalized (lowercased/stripped), and `host_tax_id` and `host_scientific_name` must both be blank or both set. Tokens not in the map are logged as warnings (not fatal) so the map can be extended over time; unparseable strain names (no 2nd token) are counted and skipped. Input metadata columns and output columns both match the `METADATA_COLUMNS` defined in `scripts/extract_ha_cds.py`.

Outputs per tree:
- `results/trees/{tree}/metadata_all_pre_host_inferred.tsv`: merged pre-host metadata with `host` / `host_tax_id` filled where the strain-token map allowed

### `annotate_host_taxonomy`
Per-subtype rule that annotates metadata with host taxonomy classification. Uses `scripts/annotate_host_taxonomy.py`. Takes the inferred-host metadata from `infer_missing_host` and the NCBI taxonomy dump. Uses `taxonkit` to look up lineage (class, order, and genus) for each unique `host_tax_id`. Renames `host` to `host_specific`, adds `host_general` (classifies as "human", "avian", "swine" (genus *Sus*), "bovine" (genus *Bos*), "equine" (order Perissodactyla), "carnivore" (order Carnivora), or "other mammal" for any other Mammalia; raises error for any non-null host outside Aves/Mammalia; deleted tax IDs with no lineage are treated as unknown with a warning), and `host_order` (taxonomic order of the host). Null/empty host tax IDs pass through as null for all host columns.

Outputs per tree:
- `results/trees/{tree}/metadata_all.tsv`: columns are `accession`, `strain`, `subtype`, `date`, `host_specific`, `host_tax_id`, `host_general`, `host_order`, `location`, `region`, `passage_history`, `length`

### `subsample`
Per-subtype rule that subsamples HA sequences using `augur subsample`. The subsampling configuration is defined per tree in `config["trees"][tree]["augur_subsample"]` and follows the `augur subsample` YAML config schema (with `defaults` and/or `samples` sections). At rule runtime the Snakefile deep-copies that config, injects `include: results/trees/{tree}/accessions_to_include_combined.txt` into every `defaults` / `samples.*` entry, and dumps the result to `results/trees/{tree}/subsample_config.yaml` which is passed via `--config`. Uses `--metadata-id-columns accession` to match the metadata format. A fixed `--seed 1` ensures reproducibility. The combined-include file and any `exclude` paths in the config are listed as explicit inputs so Snakemake tracks them as dependencies. Sequences come from `merge_manual_add` (so manually-added strains are eligible for subsampling).

Outputs per tree:
- `results/trees/{tree}/metadata_subsampled.tsv`: subsampled metadata
- `results/trees/{tree}/cds_subsampled.fasta`: subsampled HA CDS nucleotide FASTA
- `results/trees/{tree}/subsample_config.yaml`: the augur subsample YAML config used

### `collapse_host_order`
Per-subtype rule that collapses low-frequency host orders to "other" in the subsampled metadata. Uses `scripts/collapse_host_order.py`. Host orders with frequency below `config["collapse_low_freq_host_order"]` (default 0.005) in the subsampled dataset are renamed to "other". Null host orders are left as-is. Logs the orders collapsed and their frequencies.

Outputs per tree:
- `results/trees/{tree}/metadata_subsampled_collapsed.tsv`: subsampled metadata with low-frequency host orders collapsed to "other"

### `align`
Per-subtype rule that aligns subsampled HA CDS sequences using `augur align` (which wraps `mafft`). Uses a per-tree reference sequence specified in `config["trees"][tree]["reference_sequence"]` (a FASTA file in `data/`) to guide the alignment. The reference is removed from the output alignment via `--remove-reference`. Uses 4 threads.

Outputs per tree:
- `results/trees/{tree}/alignment.fasta`: aligned HA CDS nucleotide FASTA (without the reference sequence)

### `tree`
Per-subtype rule that infers a phylogenetic tree from the alignment using `augur tree` (which wraps IQ-TREE by default). Uses `--tree-builder-args='-seed 1 -czb'` for reproducibility (`-seed 1`) and to collapse zero-length branches (`-czb`). Uses 4 threads. The output is named `tree_raw.nwk` to indicate it is before any temporal refinement.

Outputs per tree:
- `results/trees/{tree}/tree_raw.nwk`: inferred maximum-likelihood tree in Newick format

### `refine`
Per-subtype rule that refines the raw tree using `augur refine` (which wraps TreeTime). Takes the raw tree, alignment, collapsed subsampled metadata, and the combined-include file from `build_include_file` as inputs. Uses `--metadata-id-columns accession` to match the metadata format. Builds a time-resolved tree (`--timetree`) using FFT-based marginal date estimation (`--use-fft`). Always passes `--keep-ids` pointing at the combined-include file so Genbank-listed and manual-add accessions are exempt from filters (their dates are inferred rather than taken from metadata if they fail the clock filter). Additional `augur refine` flags are configured per tree in `config["trees"][tree]["augur_refine"]` as a dict of flag-name to value; all entries are passed as `--{key} {value}` flags (e.g. `--clock-filter-iqd` to remove molecular clock outliers). The combined stdout+stderr from `augur refine` is written to both a Snakemake output file (`refine_output.txt`, for downstream parsing by `parse_refine_outliers`) and a Snakemake log file (preserved on failure for debugging), using `tee` with `set -eo pipefail`.

Outputs per tree:
- `results/trees/{tree}/tree.nwk`: time-resolved refined tree in Newick format
- `results/trees/{tree}/branch_lengths.json`: node data JSON with branch lengths, divergence, and inferred dates
- `results/trees/{tree}/refine_output.txt`: combined stdout+stderr from `augur refine` (used as input by `parse_refine_outliers`)

### `parse_refine_outliers`
Per-subtype rule that parses the `augur refine` output to extract any strains whose dates were reset because they were flagged as molecular clock outliers. Uses `scripts/parse_refine_outliers.py`. The script validates that the output is from a successful `augur refine` run by checking for the sentinel line `"Inferred a time resolved phylogeny using TreeTime:"`, then looks for the outlier block (`"the following tips have been marked as outliers"`). If present, it parses each entry's accession, metadata date (decimal year), and TreeTime-inferred date. Multi-line entries (where the apparent date wraps to a continuation line) are collapsed before parsing. If no outlier block is found, an empty TSV (header only) is written. Safety checks: raises an error if the sentinel is missing (incomplete run or format change), if the outlier header is found but no entries can be parsed (format change), or if the word "outlier" appears outside the known outlier block (unexpected format change). This prevents silently reporting zero outliers when the format has changed. The `bad_dates_in_keep_accessions_action` config option (passed as a param) controls behavior when outliers are found: `"ignore"` writes the TSV normally, `"error"` raises a `ValueError` listing the affected accessions and their dates. Any other value raises an error immediately.

Outputs per tree:
- `results/trees/{tree}/accessions_with_bad_dates_reset.tsv`: TSV with columns `accession`, `metadata_date`, `inferred_date` listing strains whose dates were reset as outliers (empty table if none; not written if action is `"error"` and outliers exist)

### `ancestral`
Per-subtype rule that infers ancestral nucleotide sequences using `augur ancestral`. Takes the refined tree and alignment as inputs. Uses `--root-sequence` with the same reference sequence used for alignment (`config["trees"][tree]["reference_sequence"]`) so that mutations are reported relative to that reference's coordinate system. Uses `--seed 1` for reproducibility.

Outputs per tree:
- `results/trees/{tree}/nt_muts.json`: node data JSON with nucleotide mutations on each branch

### `translate`
Per-subtype rule that translates nucleotide mutations to amino acid mutations using `augur translate`. Takes the refined tree, ancestral nucleotide mutations, and a GFF3 annotation file (`config["trees"][tree]["annotation"]`) as inputs. The `--genes` flag (`config["trees"][tree]["genes"]`) specifies which genes to translate (e.g., SigPep, HA1, HA2).

Outputs per tree:
- `results/trees/{tree}/aa_muts.json`: node data JSON with amino acid mutations on each branch

### `score_mutation_effects`
Per-subtype rule that scores tree nodes based on per-mutation phenotype effects. Uses `scripts/score_mutation_effects.py`. Takes the refined tree, amino acid mutations node data, and a mutation effects TSV file (`config["trees"][tree]["mutation_effects"]`) as inputs. The mutation effects TSV must have columns `protein`, `protein_site`, `wildtype`, `mutant`, `phenotype`, `effect`. The script validates that the set of proteins in the mutation effects file exactly matches the set of genes in the amino acid mutations node data (raises an error if not). Phenotypes to score are specified in `config["trees"][tree]["phenotypes"]` (keys are phenotype column values in the TSV, values are display names for the tree). For each phenotype, computes cumulative amino acid mutations from root to each node, then calculates three metrics: total effect (sum), max magnitude effect (mutation with largest absolute effect value), and the identity of that max magnitude mutation.

Outputs per tree:
- `results/trees/{tree}/mutation_effects_scores.json`: node data JSON with phenotype scores for each node

### `generate_phenotype_auspice_config`
Per-subtype rule that generates an auspice config JSON defining color scales for phenotype scores. Uses `scripts/generate_phenotype_auspice_config.py`. Takes the mutation effects scores node data as input. Reads min/max values for each phenotype's total effect and max magnitude effect, then creates continuous viridis color scales. The color scale hex values are configured per tree in `config["trees"][tree]["phenotype_color_scale"]`. Also adds a categorical coloring for max magnitude mutation.

Outputs per tree:
- `results/trees/{tree}/auspice_config_phenotypes.json`: auspice config JSON with phenotype color scales

### `annotate_strains`
Per-subtype rule that maps the per-strain annotations TSV (`config["trees"][tree]["strain_annotations"]`) onto tree tips. Uses `scripts/annotate_strains.py`. Takes the refined tree and the strain annotations TSV as inputs. Params are the global `config["missing_strain_annotations_action"]` and the tree's `phenotype_color_scale` (reused from the phenotype color config).

Behavior:
- If the TSV is empty (header-only or fully empty), writes `{"nodes": {}}` and `{"colorings": [], "filters": []}` and returns — nothing else happens.
- Otherwise requires an `accession` column (any position). Errors on duplicates, whitespace in `accession` values, or a missing `accession` column.
- Reads tip names from the refined `tree.nwk` via `Bio.Phylo`. For accessions not present as tips, obeys `missing_strain_annotations_action`: `"ignore"` warns and drops those rows; `"error"` raises a `ValueError` listing them; any other value errors immediately.
- Classifies each non-`accession` column as numeric (all non-null values parse as numbers) or non-numeric. Numeric columns are cast to floats and get a `continuous` auspice `coloring` with a scale built by interpolating each entry of `phenotype_color_scale` across `[min, max]` of the column's observed values. Non-numeric columns get a `categorical` coloring and are added to `filters`.

Outputs per tree:
- `results/trees/{tree}/strain_annotations_node_data.json`: node data JSON with per-tip annotation values (passed to `augur export --node-data`)
- `results/trees/{tree}/auspice_config_strain_annotations.json`: auspice config JSON with colorings (and filters for categorical columns)

### `format_description`
Per-subtype rule that formats the shared description markdown file for a specific tree. Takes the description markdown file from `config["trees"][tree]["description"]` and replaces all occurrences of `{tree}` with the tree name (e.g., "H5"). Uses `sed` in a shell command.

Outputs per tree:
- `results/trees/{tree}/description.md`: formatted description markdown with `{tree}` replaced by the tree name

### `export`
Per-subtype rule that exports an auspice JSON for visualization. Uses `augur export v2`. Takes the refined tree, branch lengths, nucleotide mutations, amino acid mutations, mutation effects scores, strain annotations node data from `annotate_strains`, collapsed subsampled metadata, an auspice config file (`config["trees"][tree]["auspice_config"]`), the generated phenotype auspice config, the generated strain annotations auspice config from `annotate_strains`, and the formatted description markdown from `format_description` as inputs. Uses `--metadata-id-columns accession`, `--include-root-sequence-inline`, `--title` (from `config["trees"][tree]["title"]`, shell-escaped with `shlex.quote`), `--description`, and `--auspice-config` (the base config plus the phenotype and strain annotations configs). The output path is `auspice/{auspice_prefix}_{tree}.json`, where `auspice_prefix` is set in `config.yaml` (typically matching the repo name for Nextstrain community builds).

Outputs per tree:
- `auspice/{auspice_prefix}_{tree}.json`: auspice v2 JSON for interactive tree visualization

### `prune_tree`
Per-entry rule (wildcard `{pruned_tree}`) that post-processes a source tree's exported Auspice JSON down to only tips whose rows in the source tree's `strain_annotations.tsv` match a column/value filter. Driven by the top-level `config["pruned_trees"]` mapping; each entry requires `source_tree` (a key in `config["trees"]`) and `keep_where` (a dict of column->value pairs; a tip is kept iff every column equals the given value after string coercion), and may specify an optional `title` to overwrite `meta.title` on the pruned JSON. Uses `scripts/prune_auspice_json.py`. Internal unary nodes left after pruning are collapsed, with the collapsed node's `branch_attrs.mutations` prepended (per gene) onto the surviving child's branch so no mutation info is lost. Errors if `keep_where` references a missing column, if no rows match, if any matched accession is not a tip in the source Auspice JSON, or if pruning removes the entire tree. The pipeline uses `wildcard_constraints` on `{tree}` (the export rule's wildcard) restricted to `config["trees"]` keys so the `{pruned_tree}` wildcard can't accidentally match an export target.

Outputs per entry:
- `auspice/{auspice_prefix}_{pruned_tree}.json`: pruned Auspice v2 JSON (listed as a `rule all` target)

### `validate_includes`
Per-subtype rule that verifies every accession specified to include (user's `accessions_to_include.txt` plus all accessions in `manual_add_metadata.tsv`) appears as a leaf in the final Auspice JSON. Uses `scripts/validate_includes.py`. The Genbank include file is parsed with `parse_accessions_file` from `scripts/build_include_file.py` (both scripts share the parser), which strips `#` comments and blank lines. For each expected accession, `source` is `manual_add` if it is in `manual_add_metadata.tsv` else `genbank` (manual_add wins on overlap). The rule walks the Auspice JSON tree, collects leaf `name` values, and writes `include_validation.tsv` with columns `accession`, `source`, `in_tree`. If any expected accession is missing from the tree, raises a `ValueError` listing the missing accessions (grouped by source), which fails the Snakemake job. The validation TSVs are targets of `rule all`, so the pipeline errors by default if any include is dropped.

Outputs per tree:
- `results/trees/{tree}/include_validation.tsv`: TSV with one row per expected include accession, columns `accession`, `source`, `in_tree`
